# Cronistoria modifiche (sintetica)

Questa cartella **non** sostituisce `TEST-SKILL/SKILL-2.0.md` né le skill Cursor: serve solo a **tracciare in modo breve** cosa è stato fatto sul repo, per chi riprende il lavoro dopo giorni o in un’altra chat.

## Come usarla

1. Aggiorna **`registro.md`** quando chiudi un intervento utile (una riga o un elenco puntato per data).
2. Non duplicare qui interi prompt o testi lunghi: basta **data, ambito, file toccati, obiettivo raggiunto**.
3. I commit Git restano la fonte dettagliata (`git log`); qui va solo il **riassunto operativo**.
